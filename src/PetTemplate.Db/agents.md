# Инструкции для работы с проектом PetTemplate

## Entity (Сущности БД)

### Конструкторы

- **ЗАПРЕЩЕНО** использовать **primary constructor** для Entity-сущностей
- Primary constructor вызывает конфликты с Entity Framework Core и DI контейнером
- Используйте обычные свойства с инициализаторами объектов

## DTO

- По возможности используем **одну модель DTO** для всех операций: `Create`, `List`, `Update`, `Delete`
- Поля, которые не используются в конкретной операции, просто игнорируем
- **Дефолтные значения допустимы:** Например, при создании сущности у нас не будет значения в поле `Created` - это нормально, пусть там будет дефолтное значение
- **Тестирование:** В тестах для операции `Update` необходимо проверять, что системные поля (например, `Created`, `CreatedBy`, `Id`) **не изменились**, даже если они передаются в метод `Update`
- Это упрощает работу с API и уменьшает количество моделей

### Связи с другими сущностями

- Рассчитываем на **умный фронтенд**
- Если в DTO есть ссылка на другую сущность (например, организацию, пользователя), достаточно иметь только **ID** этой сущности
- **НЕ добавляем** дополнительные поля типа `OrganizationName`, `UserName`, `FounderEmail` и т.д.
- Фронтенд сам получит необходимые данные по ID через соответствующие API
- Исключение: поля, которые критичны для отображения в списках и требуют минимизации запросов (решается индивидуально)
